Abstract

This paper presents a real-time deep reinforcement learning processor for mapless autonomous navigation targeting resource- and energy-constrained mobile robots. The unified actor-critic network architecture combined with feature-map caching enables parameter sharing and eliminates redundant computations. This reduces total parameter count, external memory access, and overall operations by 95.6%, 84.1%, and 85.4%, respectively, without compromising learning performance.

The processor further incorporates inference-on-request scheduling to achieve concurrent training and inference, ensuring low inference latency while maintaining 71.0% PE utilization including external I/O overhead. A column-wise zero-skipping categorical projection accelerates distributional reinforcement learning by processing only non-zero entries in the sparse Delta h matrix, achieving a 99% reduction in projection operations.

Fabricated in a 28-nm CMOS process, the chip consumes 2.68 mW at 10 MHz while satisfying real-time control requirements. The processor was verified on a real robot and demonstrated robust autonomous navigation in unseen environments.

Problem Setting

In mapless autonomous navigation, the robot must act using only onboard observations instead of a pre-built map. That means the system has to react within the sensing period, learn under strict memory and power constraints, and still remain robust in ambiguous or dynamic environments.

This motivates a joint algorithm-hardware design: the navigation policy has to be compact enough for embedded deployment, the scheduler has to prioritize latency-critical inference, and the accelerator has to support the training workload efficiently rather than only optimize peak throughput.

Key Contributions

Unified Actor-Critic Network + Feature-Map Caching

Shares the actor-critic backbone and reuses intermediate observation features to reduce redundant compute and memory traffic.

Adaptive Dataflow Mapping

Reconfigures memory-to-array dataflow across training and inference modes to keep the PE array efficiently utilized.

Inference-on-Request Scheduling

Lets background training yield immediately to latency-critical inference requests from the robot.

Accelerates categorical projection with a dedicated zero-skipping design specialized for sparse Delta h matrices.

Inference-on-Request (IoR) Scheduling

Sparse training is useful for navigation workloads, but conventional sequential scheduling creates idle periods and forces the hardware to run at unnecessarily high frequency. The proposed inference-on-request scheduling treats training as interruptible background work and lets inference preempt it whenever the robot requests an action.

At the selected inference-to-training ratio of N = 8, this reduces the required operating frequency by 86.4% relative to conventional sequential scheduling while preserving real-time response.

Overall Architecture

The processor consists of the following main components:

  • 8 x 32 processing element (PE) array
  • 16-kB Multi/Broadcast Memory (MBMEM)
  • 512-kB Multi/Unicast Memory (MUMEM)
  • 64-kB Output Memory (OMEM)
  • Instruction Management Unit (IMU)
  • Vector Processing Unit (VPU)
  • Projection Processing Unit (PPU)
  • Top-level controller with a 128-bit external I/O interface

Adaptive Dataflow Mapping

DRL workloads must support both multi-batch feed-forward execution during training and single-batch feed-forward execution during inference. The proposed processor adaptively reconfigures how MBMEM and MUMEM deliver activations and weights so the PE array remains efficiently utilized across these distinct modes.

In training, the mapping is optimized for multi-batch throughput, while in inference it flips data placement to efficiently support the single-batch path. This avoids the utilization loss that conventional row-limited multicast approaches suffer from when batch size is small.

Distributional Reinforcement Learning Support

Unlike prior DRL accelerators focused on scalar Q-value methods, this design supports distributional RL through a dedicated projection path. Since the Delta h matrix is highly sparse, with at most two non-zero entries in each column, the projection engine processes only those active entries and skips unnecessary work on zeros.

This reduces projection operations by 99% for 51 supports and makes the DistRL workload practical on a compact edge processor.
